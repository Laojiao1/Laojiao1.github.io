神经网络量化入门

神经网络量化入门

背景知识

量化常用于模型压缩与加速

虽然图片归一化和量化在数学形式上相似，但深度学习中的“量化”特指 模型参数和中间特征图(Activation) 的低位宽表示。

在深度学习中，量化指的是使用更少的 bit 来存储原本以浮点数存储的 tensor，以及使用更少的 bit 来完成原本以浮点数完成的计算。这么做的好处主要有如下几点：

  • 更少的模型体积，接近 4 倍的减少；
  • 可以更快的计算，由于更少的内存访问和更快的 int8 计算，可以快 2~4 倍。

一个量化后的模型，其部分或者全部的 tensor 操作会使用 int 类型来计算，而不是使用量化之前的 float 类型。

一般来说，我们会将一张 uint8 类型、数值范围在 0~255 的图片归一成 float32 类型数值范围在 0.0~1.0 的张量，这个过程就是反量化。

类似地，我们经常将网络输出的范围在 0.0~1.0 之间的张量调整成数值为 0~255、uint8 类型的图片数据，这个过程就是量化。

所以，量化的本质： 是对数值范围的重新调整，可以粗略的理解为一种线性映射（*有些论文会用非线性量化）。

反量化：一般没有信息损失；量化：一般会有精度损失

  • 因为 float32 能保存的数值范围本身就比 uint8 多

  • 量化模型和全精度模型的误差就来自四舍五入的 clip 操作。

    这篇文章中会用到一些公式，这里我们用 rrr 表示浮点实数，qqq 表示量化后的定点整数。浮点和整型之间的换算公式为：

r=S(q−Z)r = S(q-Z) r=S(q−Z)

q=round(r/S+Z)q = round(r/S + Z) q=round(r/S+Z)

  • S 是 scale，表示实数与整数之间的比例关系
  • Z 是 zero point，表示实数中的 0 经过量化后对应的整数

S=rmax⁡−rmin⁡qmax⁡−qmin⁡S = \frac{r_{\max} - r_{\min}}{q_{\max} - q_{\min}} S=qmax​−qmin​rmax​−rmin​​

Z=round⁡(qmax⁡−rmax⁡S)Z = \operatorname{round} \left( q_{\max} - \frac{r_{\max}}{S} \right) Z=round(qmax​−Srmax​​)

需要强调的一点是，定点整数的 zero point 就代表浮点实数的 0，二者之间的换算不存在精度损失**（对称量化）**

  • 这么做的目的是为了在 padding 时保证浮点数值的 0 和定点整数的 zero point 完全等价，保证定点和浮点之间的表征能够一致。

关于对称量化和非对称量化

对称量化（Symmetric Quantization）

  • 特点：Z = 0；r = S(q−Z)

  • 也就是：整数 0 直接对应浮点 0

    数值范围长这样：

关于 0 完全对称。

非对称量化（Asymmetric Quantization）

  • 特点：Z ≠ 0；r = Sq

  • 也就是：浮点 0 不一定对应整数 0

    数值范围：

不要求关于 0 对称。

为什么会有这两种？

  • 因为神经网络里的数据分布不同。
  • 通常来说：
    • 对称量化适合卷积核，因为它通常是对称的
    • 非对称量化适合激活值，因为它通常是不对称的，尤其是经过 ReLU、SiLU 等激活函数后，大量的数据都是正数，如果强行量化，会导致负半轴完全浪费，损失大量精度

矩阵运算的量化

量化矩阵乘的本质：用整数完成所有乘加运算，最后用一个缩放系数恢复到浮点结果

由于卷积网络中的卷积层和全连接层本质上都是矩阵乘法，因此先看如何将浮点运算上的矩阵转换为定点运算。

（1）首先，原始矩阵乘法（浮点）是：r3i,k=∑r1i,j⋅r2j,kr_3^{i,k} = ∑r_1{i,j}⋅r_2{j,k}r3i,k​=∑r1​i,j⋅r2​j,k

但量化之后，我们不再直接用浮点数r，而是用整数q，通过：r=S(q−Z)r = S(q-Z)r=S(q−Z)

也就是说：

  • qqq：整数（int8）
  • SSS：scale（缩放系数）
  • ZZZ：zero-point（零点）

这是量化的本质：用整数 + 缩放还原浮点

（2）其次，把矩阵乘“换成整数版本”

Step1：

  • 把每个 r 换成量化表达：r1=S1(q1−Z1)r1=S1(q1−Z1)r1=S1(q1−Z1) r2=S2(q2−Z2)r2=S2(q2−Z2)r2=S2(q2−Z2)
  • 带入矩阵乘：r3=∑S1(q1−Z1)⋅S2(q2−Z2)r_3=∑S_1(q_1−Z_1)⋅S_2(q_2−Z_2)r3​=∑S1​(q1​−Z1​)⋅S2​(q2​−Z2​)

Step2：

  • 把 scale 提出来：r3=S1S2∑(q1−Z1)(q2−Z2)r_3=S_1S_2∑(q_1−Z_1)(q_2−Z_2)r3​=S1​S2​∑(q1​−Z1​)(q2​−Z2​)

**Step 3：**再把输出也量化：

  • 输出同样满足：r3=S3(q3−Z3)r3=S3(q3−Z3)r3=S3(q3−Z3)
  • 于是两边相等：S3(q3−Z3)=S1S2∑(q1−Z1)(q2−Z2)S3(q3−Z3)=S_1S_2∑(q_1−Z_1)(q_2−Z_2)S3(q3−Z3)=S1​S2​∑(q1​−Z1​)(q2​−Z2​)

**Step 4：**整理得到最终公式：

  • q3=S1S2SS3∑(q1−Z1)(q2−Z2)+Z3q3=\frac{S_1 S_2}{S_{S_3}}∑(q_1−Z_1)(q_2−Z_2)+Z_3q3=SS3​​S1​S2​​∑(q1​−Z1​)(q2​−Z2​)+Z3​

令 M=S1S2S3M = \frac{S_1 S_2}{S_3}M=S3​S1​S2​​，其中 MMM 是一个 (0,1)(0, 1)(0,1) 之间的浮点数。

硬件上通过 Multiplier (整数) 和 Shift (右移) 来近似：$$M \approx M_0 \cdot 2^{-n}$$

这样，整个推理过程就变成了：累加结果(int32)∗M0>>n+Z3累加结果(int32) * M_0 >> n + Z_3累加结果(int32)∗M0​>>n+Z3​。这就是量化推理比浮点快几倍的底层原因。

这个公式意在表达：量化矩阵乘 = 整数乘加 + 一个缩放因子

这个公式干了什么？

  • 去掉 zero-point（中心平移），因为整数的“0”不一定对应浮点的“0”
  • 用整数做乘法和加法，这一部分完全是整数运算
  • 最后补一个缩放，把结果“拉回正确尺度”

比如：

  • 原始浮点版本 float × float → float
  • 量化版本 int × int → int（累加） → 再乘一个scale → 得到结果

一个具体的例子：

浮点世界（真实世界）：3.2 × 1.5 + 2.1 × 0.7

本质就是：先粗略算（整数），最后校正（scale）

补充：

在实际的工程实现中：不会每一步都乘 scale，而是：

  • 先 int8 × int8 → int32 累加
  • 最后一次性乘 scale

这是因为：避免精度损失 + 提升速度

此外，在矩阵量化中我们通常用一个 Scale，但在特殊情况下，比如不同卷积核的数值分布差异巨大，统一量化会导致某些 Channel 精度严重丢失，所以引出了 Per-tensor 和 Per-channel。

  • Per-tensor： 整个卷积层共用一个 SwS_wSw​。
  • Per-channel： 每个卷积核（Filter）拥有独立的 SwS_wSw​。

实际部署中，weight 通常采用 per-channel 对称量化，即每个输出通道独立使用 scale，以减小量化误差。

卷积网络的量化

有了上面矩阵乘法的量化，我们就可以进一步尝试对卷积网络的量化（实际就是矩阵量化的换皮）。

假设一个这样的网络：

这个网络只有三个模块，现在需要把 conv、relu 、fc 量化。

（1）统一量化表达（全局基础）

  • 整个网络所有张量统一满足：r=S(q−Z)r = S(q - Z)r=S(q−Z)

  • 后续所有计算，本质都是：通过 re-scale 完成不同量化域之间的数值映射

    • 即：int32累加结果 → 重新映射到新的int8量化域

    （2）Conv层量化（核心计算层）

  • 输入、权重分别量化：

    • 输入：(Sx,Zx,qx)(S_x, Z_x, q_x)(Sx​,Zx​,qx​)
    • 权重：(Sw,0,qw)(S_w, 0, q_w)(Sw​,0,qw​)（对称量化）

  • 通过之前的矩阵运算量化过程的推导可以算出卷积计算：

    • qconv=SxSwSconv(∑(qx−Zx)qw+bq)+Zconvq_{conv} = \frac{S_x S_w}{S_{conv}} \left( \sum(q_x-Z_x)q_w+b_q \right) +Z_{conv}qconv​=Sconv​Sx​Sw​​(∑(qx​−Zx​)qw​+bq​)+Zconv​

  • 关键点压缩：

    • int8 × int8 → int32 累加

    • 权重 Zw=0Z_w = 0Zw​=0（减少计算）

    • Scale 合并为一个常数 M（multiplier）

    • Bias 处理：bq=bSxSwb_q = \frac{b}{S_x S_w}bq​=Sx​Sw​b​，直接加在 int32 累加结果上

（3）ReLU层量化（通常采用非对称量化，几乎0成本量化）

  • ReLU：y=max⁡(0,x)y = \max(0, x)y=max(0,x)
  • 若 activation 使用非对称量化：r=S(q−Z)r = S(q - Z)r=S(q−Z)
    • 则浮点 0 对应整数 ZZZ
    • 因此：qy=max⁡(qx,Z)q_y = \max(q_x, Z)qy​=max(qx​,Z)
  • 即：ReLU 可以直接在整数域完成，本质是将小于零点 ZZZ 的整数值截断为 ZZZ。

（4）FC 层（全连接层）量化（本质是矩阵乘）

  • 完全复用前面的结论：qfc=SinSwSout∑(qin−Zin)qw+Zoutq_{fc} = \frac{S_{in} S_w}{S_{out}} \sum (q_{in} - Z_{in}) q_w + Z_{out}qfc​=Sout​Sin​Sw​​∑(qin​−Zin​)qw​+Zout​

  • 和 Conv 的唯一区别是没有空间卷积结构

  • 所以可以理解为：Conv量化 = FC量化 + 空间结构

（5）整体数据流（总结）

整个网络的量化推理流程：

scale 的传递关系：每一层都有自己的 scale

但注意：

  • ReLU 本身不涉及乘法缩放，但实际部署中其输出 activation 可能重新统计量化参数。
  • Conv / FC 的 int32 累加结果对应 scale 为 SinSwS_{in}S_wSin​Sw​，实际部署时通常需要通过 requantization（重缩放）重新映射到输出 activation 的量化 scale SoutS_{out}Sout​。（量化对齐）

补充：关于量化对齐：把当前层计算出来的数值，重新映射到下一层需要的量化范围

这是量化推理里非常核心的一步，因为 Conv / FC 的累加结果和下一层 int8 的数值范围通常不一致，所以必须“重新对齐”。

  • 想象一下：假如当前层进行了int8 * int8 → int32累加，得到了一个“大范围”的整数，但下一层只能接受int8，所以必须重新压缩到int8范围，这一步就叫做量化对齐（他做了不同量化域之间的统一）

    为什么会出现这个问题？

    卷积量化时：

  • 输入：x=Sx(qx−Zx)x = S_x(q_x-Z_x)x=Sx​(qx​−Zx​)

  • 权重：w=Swqww = S_w q_ww=Sw​qw​

  • 卷积累加：∑(qx−Zx)qw\sum(q_x-Z_x)q_w∑(qx​−Zx​)qw​

  • 它对应的真实 scale 是：SxSwS_xS_wSx​Sw​

    所以：卷积后的 int32 accumulator 对应的真实数值 scale 为：Sacc=SxSwS_{acc}=S_xS_wSacc​=Sx​Sw​

    但问题来了：下一层（比如 ReLU / FC）希望输入的是：int8 activation，而且它有自己的 scale：SoutSoutSout，而通常：SxSw≠SoutS_xS_w \neq S_{out}Sx​Sw​=Sout​，所以：当前层输出的“数值单位”和下一层要求的“数值单位”不一致，这就需要：量化对齐。

    如何对齐？

    对齐的本质就是“单位换算”，这一步其实很像：米 → 厘米

    量化里则是：当前 scale → 下一层 scale：M=SxSwSoutM=\frac{S_xS_w}{S_{out}}M=Sout​Sx​Sw​​

    然后：qout=M⋅qacc+Zoutq_{out}=M\cdot q_{acc}+Z_{out}qout​=M⋅qacc​+Zout​，其实就是之前推导出来的公式：

  • qout=SxSwSout(∑(qx−Zx)qw+bq)+Zoutq_{out} = \frac{S_x S_w}{S_{out}} \left( \sum(q_x-Z_x)q_w+b_q \right)+Z_{out}qout​=Sout​Sx​Sw​​(∑(qx​−Zx​)qw​+bq​)+Zout​

  • 更严谨一点的版本：qout=clamp(round(SxSwSout(∑(qx−Zx)qw)+bq)+Zout)q_{out} = clamp(round(\frac{S_x S_w}{S_{out}} \left( \sum(q_x-Z_x)q_w)+b_q \right)+Z_{out})qout​=clamp(round(Sout​Sx​Sw​​(∑(qx​−Zx​)qw​)+bq​)+Zout​)

（6）工程实现本质

整个网络量化后，本质变为：一条纯整数计算图 + 每层一次重缩放，即：

  • 所有乘加 → int32 完成（由于卷积/矩阵乘会进行大量累加，若仍使用 int8 容易溢出，因此乘法结果通常在 int32 域中累加。）
  • 所有非线性 → 整数域完成
  • 所有精度恢复 → scale 完成

对于 x→Conv→ReLU→FC→y 网络，量化的核心是：将各层的浮点计算统一映射为整数域中的乘加与截断操作，并通过 scale 和 zero-point 在层间进行数值范围对齐，从而实现高效推理同时尽量保持精度。

总结

本文从最基础的量化公式出发，逐步推导了矩阵乘法的量化实现，并进一步扩展到了卷积网络中的 Conv、ReLU、FC 等核心模块，完整梳理了神经网络量化推理的基本流程。

整个量化体系的核心，可以概括为一句话：用整数近似浮点，用 scale 恢复数值。

具体来说：

  • 浮点 tensor 被映射为 int8 等低位宽整数；
  • 卷积与全连接中的乘加运算统一转为：
    • int8 × int8 → int32 accumulation
  • 非线性激活（如 ReLU）直接在整数域完成；
  • 最后通过 requantization（重缩放）完成不同量化域之间的数值对齐。

本文重点解释了几个量化中的核心概念：

  • scale 与 zero-point 的物理意义
  • 对称量化与非对称量化
  • 矩阵乘法如何改写为整数乘加
  • 为什么需要 int32 accumulator
  • 为什么需要 requantization（量化对齐）
  • Per-tensor 与 Per-channel 的区别
  • Conv / FC / ReLU 的量化实现逻辑

可以看到：神经网络量化并不是简单地“把 float 换成 int”，而是重新建立了一套整数域中的数值表示体系。

其本质是：

  • 不同量化域之间的数值映射
  • scale 的传播与对齐
  • 精度与计算效率之间的权衡

下一章节预告

在本文中，为了便于理解量化流程，我们直接手动指定了：scale / zero-point

但在真实工程部署中：量化参数的选择，往往比量化公式本身更加重要。

因为：

  • scale 过大 → 精度下降严重
  • scale 过小 → 容易发生截断与饱和
  • 不同层的数据分布差异巨大
  • activation 往往存在长尾分布
