动态量化与静态量化

动态量化与静态量化

很多人第一次学量化时会认为 “量化不就是提前算好 scale，然后推理时直接 int8 吗？”

但实际上：量化最大的难点之一恰恰是 —— activation 的范围是不稳定的

因为 weight 是固定的，但 activation 是跟输入数据强相关的。

比如对于同一个模型来说，输入白天图像和输入夜晚图像的 activation 分布可能完全不同。

于是问题就来了：scale 到底应该什么时候统计？

这就引出了：动态量化（Dynamic Quantization）与 静态量化（Static Quantization）

动态量化（Dynamic Quantization）

动态量化的核心思想非常直接：推理时，现场统计 activation 范围。

也就是说：模型运行过程中，每次输入数据时，都会重新计算：

  • min/max
  • scale
  • zero-point

所以，动态量化最大的特点是：scale 是实时变化的。

这样做最大的好处是：activation 范围适应性极强，因为每次都用当前数据的真实分布，不容易 overflow，scale过大，不容易动态范围浪费，因此动态量化通常精度较高。

但与此同时，它的问题也比较明显：推理时需要额外统计，资源耗费大，速度较慢。

想象一下，你在每次推理时都要：遍历 activation、统计 min/max、计算 scale、计算 zero-point，那么这一定会产生：额外 latency、更多 memory access、以及更多 CPU/GPU 开销。所以，动态量化虽然精度不错，但速度不一定最快，特别是在 DSP 或 NPU 上。

硬件最怕动态变化，它最喜欢的是提前固定好

此外，动态量化还有一个重要的特点：它通常只量化 weight

静态量化（Static Quantization）

静态量化则完全相反，它的思想是：推理前，提前把所有的 scale 固定好（上节课我们就是用的这种方式）

也就是说，在部署之前，先拿一批 calibration dataset 跑一边模型，统计：

  • activation histogram
  • min/max
  • percentile
  • KL
  • MSE

然后提前确定每一层的 scale。在部署后和推理时，直接使用固定的 scale，不再动态计算。

这样做最大的好处是：推理极快。

  • 原因显而易见：不用再统计分布，更新scale，计算zero-point，整个计算图直接变成纯整数流，对硬件极为友好，因为硬件终于可以提前优化整个整数计算图了。

但静态量化最大的问题也很明显：

  • 因为 scale 一旦固定，后面将永远不变。于是，如果 calibration 数据不能代表真实数据分布，那么部署后 activation 很可能直接炸掉。
  • 例如校准集全是白天图像，结果部署后来了夜间图像，于是 activation 范围变化巨大，这时候固定的 scale 可能就会出现很多问题，导致精度暴跌。

这也是为什么很多人发现 TensorRT INT8 离线测试很好，但线上突然崩。因为问题往往不在模型，而是 calibration dataset 不够真实

所以 动态量化 vs 静态量化 本质其实是 —— 灵活性 vs 性能之间的 trade-off。

方法 scale 精度 速度 硬件友好
动态量化 实时计算 高 较慢 一般
静态量化 提前固定 依赖校准 极快 极强

现代量化研究，其实一直都在努力做一件事：用静态量化，逼近动态量化的精度。

实际工程中，是怎么做的？

在实际工业界（如车载芯片、手机端侧 NPU、服务器 GPU）中，两者的落地逻辑完全不同：

静态量化

静态量化在真实工程中，绝对不是在线跑模型时去算参数，而是采用完全离线交接的逻辑：

  1. 离线校准（Offline Calibration）：在开发阶段，拿 100~500 张真实校准集图片跑前向传播，利用 KL 或 MSE 算法在后台生成一个密密麻麻的文本文件 —— 校准缓存表（Calibration Cache Table），锁定每一层的固定 Scale。
  2. 图层编译（Graph Compilation）：将原始浮点模型和校准表同时喂给推理引擎（如 TensorRT），编译器会通过模式匹配（Pattern Matching），把图里用来仿真的 Div/Round/Mul 节点物理擦除，将固定的 Scale 常量直接强行灌入硬件寄存器。
  3. 线上纯整数流（Runtime）：模型在线上跑实时业务时，离线计算早已结束。硬件不再做任何多余统计，直接开启纯 INT8 的硬件级极速矩阵乘加，将 Latency 降到极致。

动态量化

很多人好奇：既然动态量化有内存访问延迟大、算力浪费的缺点，为什么工业界还没淘汰它？

万物都有其存在的道理：

动态量化是大语言模型（LLM）的救星：在 Transformer 架构（如 Llama、ChatGPT）中，线性层的权重（Weight）是极其庞大的，但其实每次推理时的 Token（Activation）相对较小，此时最核心的瓶颈是带宽拉取权重的速度（Memory Bound），并不是计算速度，所以动态量化非常契合大语言模型的特点。

特定策略（Weight-Only / SmoothQuant）：工业界常在 LLM 中用动态量化只量化权重（INT8/INT4），并在运行时动态把权重解压回浮点与激活值计算。既压缩了模型体积、减少了带宽占用，又完美避开了激活值现场统计带来的 Latency 雪崩。
