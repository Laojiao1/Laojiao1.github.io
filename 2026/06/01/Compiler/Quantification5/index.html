还记得我们之前在《2.1 神经网络量化入门》里跑过的那个极简的神经网络吗？

当时的结构是标准的： x -> conv -> relu -> fc -> y。

但由于那时我们图省事，直接人为指定了量化参数，导致最终的量化输出与原始 FP32 结果之间产生了极其巨大的精度误差。

这就是量化工程落地最真实的写照：不进行真实数据统计，数值分辨率就会大面积坍塌，精度就会彻底崩盘。

本节笔记的核心任务，就是用我们这几节学到的“量化校准（Calibration）”技术，彻底捞回这个小网络的精度。我们将真刀真枪地在 PyTorch 框架里完整重建原有的 NumPy 神经网络，通过把真实的校准集数据（Calibration Dataset）喂给模型，离线“拦截”并统计出每一层真实的激活值 Histogram 与 Min-Max 范围。

此外，在这场实战中，我们还将直面矩阵乘法形状不匹配（Shape Mismatch）等硬核工程坑，最终端到端导出一份让量化误差无限逼近于 0、兼具数学合法性与硬件加速能力的完美 ONNX 模型资产

本章结语

到现在为止，我们已经跨过了量化领域的三座大山，让我们一起来回顾一下学过的内容：

一、从数学公式到数值分辨率（入门与对齐篇）

  • 核心理解：搞懂了 r=S(q−Z)r = S(q - Z)r=S(q−Z) 这个看似简单的映射公式背后，真正决定精度的是 Scale（比例尺）
  • 分辨率博弈：深刻理解了 Scale 越大，一个整数格子代表的浮点跨度就越大，精度（分辨率）就越粗糙。量化的本质不是“转类型”，而是在动态范围（防止溢出）与数值分辨率（保留有效位）之间做系统级的 Trade-off

二、五大统计基石与校准（算法篇）

不再盲目拍脑袋定参数，而是看清了四大校准算法的哲学博弈：

  • Histogram 统计：量化的“天眼”。用离散区间（Bins）近似描述海量激活值的概率密度，暴露出长尾分布的“案发现场”。
  • Min-Max：激进的完美主义者。绝不截断，但极度害怕离群值（Outliers），极易为了几个无辜的极端异常值而稀释整个网络的主体分辨率。
  • Percentile：工程上的实用主义者。通过砍掉 0.01%0.01\%0.01% 的极端值，瞬间换取主体区域几十倍的分辨率提升。
  • KL Divergence：统计学大师（TensorRT 招牌）。不看单点数值，只关心量化前后的概率分布保真度，通过最小化信息熵损失，让语义结构不发生畸变。
  • MSE：暴力的死磕派。通过穷举截断点，直接最小化量化前后的均方误差（数值保真度）。

三、多框架精度差异与系统工程（部署篇）

跨越“纸上谈兵”阶段，在真正的工程对齐中，看透了不同后端（TensorRT / TFLite）精度差异的隐蔽根源：

  • 定点乘法近似：硬件底层不爱浮点，通过 M≈M0×2−nM \approx M_0 \times 2^{-n}M≈M0​×2−n 的整数乘法与位移细节逼近缩放因子。
  • 算子融合（Operator Fusion）时机：在 Conv + BatchNorm + ReLU 融合前还是融合后做直方图统计，截获的数据分布天差地别。
  • 混合精度与敏感度分析：通过单点 Fallback 策略，将拖累精度的极少数关键敏感层回退到 FP16/FP32 运行，用局部的速度妥协换取整体精度的“工程可用”。

然而，旅途仅仅刚刚开始……

当我们在 Netron 中看到自己白盒构建的 [QuantizeLinear -> DequantizeLinear] 完美对齐计算图时，意味着我们在经典 CNN/RNN 以及轻量化网络上的量化修行已经圆满

但在软硬件协同演进的宏大叙事里，前方还有更新的战场：

  • 大语言模型（LLM）的超低比特激战：面对 INT4/INT3 时代无法裁剪的强离群通道（Emergent Features），参透 SmoothQuant、AWQ、GPTQ 算法是如何重新定义权重与激活值的数学平衡
  • 量化感知训练（QAT）的梯度穿越：探索 STE（直通估计器）是如何在反向传播中让梯度跨越不可导的 Round 算子，在训练期主动让权重适应低比特噪声

革命尚未成功，同志还需努力！
